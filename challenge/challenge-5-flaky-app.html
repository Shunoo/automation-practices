The Flaky App

Challenge 5 — An intentionally unreliable app for retry logic testing

Flakiness Configuration

1. The Slow Element Timeout Flake
Click the button. A data element will appear after a random delay (500ms to max-delay). Sometimes it takes so long your test will timeout. Your retry logic should handle this.
✅ Data loaded: { users: 42, status: "ok" }
2. The Ghost Element Not-Found Flake
This element randomly appears and disappears. Trying to interact with it when it's gone causes an element-not-found error. It re-renders every 2 seconds.
🎯 Click me! I'm here... for now.
Successful clicks: 0
3. The Unreliable Counter Assertion Flake
This counter should increment by 1 each click, but sometimes it jumps by 2 or 3. An assertion error (wrong value) should NOT trigger a retry — only timeout/not-found should.
0
Expected next: 1
4. The Unreliable API Mixed Flakes
Simulates an API call that can fail with: timeout (data takes too long), not-found (endpoint temporarily unavailable), or return wrong data (assertion failure). All three failure types possible.
    5. The Flaky Form Mixed Flakes
    Fill in the form and submit. The submit button may disappear (not-found), the confirmation may take forever (timeout), or the success message may show wrong data (assertion).
